Version 3.7

15 April 2009

When the mouse glides over the Check menu, the References Window no longer opens unexpectedly.

A serious security hole was fixed.

Windows can either be attached or detached. Setting in Preferences.

By default windows are attached.

A new USFM to OSIS converter was written.

Some help was added for selecting the language for a Sword module.

Once a month the local git repository will be cleaned up, pruned and repacked for better performance and less disk usage.

The Shona language was added for the book names and their abbreviations.

A remote repository setup assistant replaced the git setup dialog. Remote repository needs to be setup again by the user, because some may cease to work.

The user can ask Bibledit to go to the next or to the previous spelling mistake.

The merge window shows the differences between the two projects at the character level.

Updated revert dialog: Less clutter, and changes show at the character level.

Any comparison function that outputs pdf compares at the character level.

The on-screen changes viewer shows the number of modifications.

The stylesheets are stored in xml format, no longer in a sqlite database.

A comment can be added when backing up.

The name of the person who created a backup is included with the backup itself.

The selected books in menu Check / Passages is limited to the relevant ones only.

The references produced by menu Check / Passages come in the references area as well.

Improvement in raw text recognition on import.

Better alignment of the verses in the Check / Passages web page output.

About a dozen of bugs were fixed.

The text area assumes the full size of the window when all other areas are empty.
