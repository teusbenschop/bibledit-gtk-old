Version 3.3

18 June 2008

Expanded dictionary editor.

Can print a book with each verse starting a new paragraph.

Styles in OpenOffice have the marker prefixed to the name, e.g. "v Verse number".

If multiple books are exported to OpenDocument, it asks whether to save them to multiple files.

It multiple references are selected in the reference area, then all of these are displayed in the quick reference area.

New check: Sentence structure.

Word count in editable dictionary.

A project note can be deleted by right-clicking on it.

The arrow keys can navigate between notes in the editor.

Unified print dialog at menu File / Print.

Diagnostics button in Help / System log.

Can print several ranges of verses in one go.

The date since which to view changes is initialized better.

Can import raw text.

Gecko does not behave well on Fedora Core 9, Macintosh, OLPC, etc., therefore GtkHtml3 is used instead.

Compiles on Fedora Core 9.

The resource viewer uses a html cache allowing for faster retrieval of remote data and offline browsing.

While merging changes, every change can be approved of by the user.

Improved dialog for importing raw text.

The status report uses a legend for the tasks so as to save space on paper.

The tasks of a range of books or chapters or verses can be set in the status editor.

The reference discovery mechanism was made more robust.

When a footnote or xref is clicked on, the screen scrolls to it properly rather than jumping to the very end of the screen.

A tool was introduced that can transfer all relevant project notes to the text, allowing for printing of project notes together with the text on one sheet.

The git repository is cleaned and pruned automatically once in 30 days, providing great savings in disk space.

When the cursor is on a heading or title, the verse number is taken from the following line, instead of the previous one.
